Use the API

        
Integrate directly with POST /api/extract in your app.

      

    

  


  
  

    
    

      Both endpoints auto-discover all tabs in the spreadsheet.
      Each row is stamped with its source sheet so you can tell them apart.
    


    
    

      1  ·  GET /parse-sheet — flat array, all tabs merged
    

    

GET /parse-sheet?sheet_url=<google_sheets_url>

// Each row has __sheet (tab name) and __gid (tab id) added:
[
  { "__sheet": "Inventory", "__gid": "0",      "product": "Sugar", "qty": "50" },
  { "__sheet": "Orders",    "__gid": "812345", "order_id": "1",  "qty": "10" }
]
    


    
    

      2  ·  POST /api/extract — grouped by tab with metadata
    

    

POST /api/extract   Content-Type: application/json
{ "url": "https://docs.google.com/spreadsheets/d/..." }

// Response — each tab is a separate object in sheets[]:
{
  "success": true, "sheetId": "1BxiMV...",
  "totalSheets": 2, "totalRows": 42,
  "sheets": [
    {
      "sheetName": "Inventory", "gid": "0", "rowCount": 20,
      "headers": ["product", "qty"],
      "data": [{ "product": "Sugar", "qty": "50" }, ...]
    },
    {
      "sheetName": "Orders", "gid": "812345", "rowCount": 22,
      "headers": ["order_id", "qty"],
      "data": [{ "order_id": "1", "qty": "10" }, ...]
    }
  ]
}
    


    

      💡 Append #gid=NUMBER to the URL to extract a single specific tab only.

      Both endpoints also work under /api prefix: /api/parse-sheet & /api/extract.
    

  


  
Sheets → JSON Extractor  ·  No data is stored. Sheets are fetched on demand.
